Introdução

Bancos de Dados Orientados a Documentos são uma das principais categorias dos Bancos de Dados NoSQL (Not only SQL) quem vêm se tornando muito populares ultimamente. São projetados para armazenar documentos de dados semi estruturados, normalmente em formato JSON ou XML.

Bancos de Dados Orientados a Documentos utilizam obviamente documentos para armazenar e consultar dados. Quando ouvimos a palavra "documento", automaticamente pensamos em algo no formato .docx ou .pdf, mas na realidade de Bancos de Dados, documentos se referem a blocos JSON ou XML. Ao invés de colunas com nomes e tipos de dados que são utilizados no Banco de Dados Relacional, o Orientado a Documentos contêm uma descrição do tipo de dado e o valor para aquela descrição. Essa estrutura pode variar de documento para documento. Documentos são agrupados em collections, que possuem uma função similar ao da tabela relacional.

Esse artigo é sobre o Cloudant, Banco de Dados Orientado a Documentos, que pertence a IBM. É uma tecnologia teoricamente recente, mas que a cada dia mais vai se tornando mais popular no mercado. O objetivo do artigo é mostrar informações sobre Cloudant, suas origens, arquitetura, formato de interação e exemplos de códigos.


Sobre o Cloudant

O Cloudant é um Banco de Dados Orientado a Documentos da IBM que inicialmente foi projetado para cloud computing (computação em nuvem), distribuindo dados entre múltiplos servidores além de redimensionar o banco de dados para que se acomode a aplicações web, através do framework do CouchDB. É também classificado como um Banco de Dados NoSQL.

Os documentos do Cloudant são em formato JSON, e ele garante que o fluxo de dados entre a aplicação e o seu Banco de Dados permaneça sem interrupção e que tenha uma alta performance.

Logo do Cloudant

Os três pilares do Cloudant:

  • Redução de tempo de inatividade
  • Mantêm os dados mais próximos aos usuários
  • Foca no desenvolvimento, não na administração do Banco de Dados

Origem

O Cloudant foi criado por três alunos que se conheceram no departamento de física do MIT (Massachusetts Institute of Technology): Alan Hoffman, Adam Kocoloski e Michael Miller. Eles trabalharam juntos em projetos sobre conjuntos de dados. Em 2008, suas idéias inovadoras para Big Data despertaram o interesse da aceleradora Y Combinator, do Vale do Silício. O resultado disso foi um investimento inicial de US $20 mil. Posteriormente, em agosto de 2010, a Avalon Ventures decidiu investir US $1 milhão na idéia, criando uma empresa chamada Cloudant.

Alan Hoffman e Adam Kocoloski, dois dos criadores do Cloudant

Ainda em agosto de 2010, a Cloudant já oferecia serviços de suporte, consultoria e treinamentos. O primeiro produto com foi lançado no final de 2010, e logo de início já era considerada pelo site CRN como uma das 10 melhores Startups Open-Source, assim como era uma idéia bem reconhecida na comunidade local de startups em Boston, Massachusetts, tendo bastante destaque em revistas de tecnologia do estado. Em janeiro de 2011, já haviam mais de 2500 clientes que utilizavam os serviços de armazenamento da Cloudant.

Michael Miller, outro envolvido no início do projeto Cloudant

Em fevereiro de 2014, a IBM anunciou um acordo para adquirir a Cloudant. O acordo foi fechado em março do mesmo ano, onde a Cloudant foi adicionada ao Grupo de Informações e Análise da IBM.

Arquitetura

O Cloudant adiciona uma determinada quantidade que elementos a um serviço que é criado em cima do Apache Lucene, que trabalha com busca por texto indexado, segurança e processamento incremental em larga escala. O objetivo do Cloudant é fazer com que o desenvolvimento de aplicações fique mais fácil. A figura abaixo mostra como o sistema se comunica em uma API RESTful, ao utilizar um navegador ou uma aplicação móvel.

Interação

É possível interagir com o Cloudant através da linguagem Python, por meio de uma biblioteca chamada Cloudant NoSQL DB Client Library.

Exemplos de códigos

A sintaxe utilizada para consultas no Cloudant é muito similar a do MongoDB, mas em um sistema totalmente novo. Abaixo podemos ver exemplos da interação do Cloudant com o Python:

Importações e estabelecimento de uma conexão:

from cloudant.client import Cloudant
from cloudant.error import CloudantException
from cloudant.result import Result, ResultByKey

client = Cloudant("", "", url="")
client.connect()

Definindo a variável, criando o Banco de Dados e confirmando se foi criado:

databaseName = ""

myDatabase = client.create_database(databaseName)

if myDatabase.exists():
print "'{0}' successfully created.\n".format(databaseName)

Deletar o Banco de Dados:

try :
client.delete_database(databaseName)
except CloudantException:
print "There was a problem deleting '{0}'.\n".format(databaseName)
else:
print "'{0}' successfully deleted.\n".format(databaseName)

Desconectar:

client.disconnect()

Conclusão

Podemos concluir que o Cloudant é uma boa opção para Banco de Dados Orientado a Documentos, entre as diversas outras opções que estão no mercado, pois aplica o conceito de Cloud Computing.
